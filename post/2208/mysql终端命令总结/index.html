创建表格#

create table userinfo(
  id int not null AUTO_INCREMENT,
  name varchar(100) not null,
  lasttime int ,
  primary key (id)
);

  • 创建远程访问的用户

    • 创建用户 create user test identified by '123456';

    • 创建数据表 create table_name;

    • 授权 gran all privileges on ``table_name``.* to Luke@'%' identified by 'ssss6666';

  • 删除unique key

    • alter table table_name drop index index_name;

  • 新增unique key

    • alter table table_name add unique key new_index_name (col1,col2);

  • 查看表的结构

    • desc table_name显示表结构，字段类型，主键是否为空等。但是不显示外键

    • show create table table_name 显示创建表的DDL

  • 修改表的储存引擎ALTER TABLE account ENGINE=MyISAM;

修改表 ALTER TABLE#

  • 修改表名 ALTER TABLE <表名> RENAME <新表名>

    • 例:ALTER TABLE game_account RENAME account;

  • 添加字段ALTER TABLE <表名> ADD <字段名称> <字段定义> [ FIRST | AFTER col_name]

    ALTER TABLE account ADD Game_zone VARCHAR(20) NOT NULL DEFAULT "HuaBei" FIRST;

    • 如果需要指定加在某个位置，在语句最后加上FIRST 或者 AFTER col_name

  • 删除字段ALTER TABLE <表名> DROP <字段名称>

    ALTER TABLE account DROP Account_role ;

  • 修改字段ALTER TABLE <表名> CHANGE <旧字段名称> <新字段名称> <字段定义>

    ALTER TABLE account CHANGE password password VARCHAR(60) ;

  • 修改表的编码 ALTER TABLE <表名> character set utf8mb4;

    alter table email character set utf8mb4;

  • 修改数据库的编码 ALTER database <库名> character set utf8mb4;

存储时间字段#

  • varchar: 存放yyyy-MM-dd HH:mm:ss 的时间格式字符串

  • datetime: 存放与上面格式一致的时间，与varchar的区别在于，它对时间做了校检，可以避免写入2月31日这样的无效日期

  • timestamp：存放时间戳，默认是10位数的（timestamp(0))，精确到秒。然后如果想存储毫秒，纳秒需要把长度对应地加大。毫秒13位长度，timestamp(3)，更精细的timestamp(6)

字段长度问题#

  • 整数类型：显示宽度和数据的取值范围没有关系，如果超过了显示宽度依然可以正常插入。

    不设置宽度，系统将添加默认的宽度 tinyint(4),smallint(6),mediumint(9),int(11),bigint(20)

  • 字符串类型：varchar(20) 指的是字符串的最大长度，如果输入了一个21长度的字符串则会丢失信息。由于varchar是变成存储的，所以开发过程中通常设置宽度为255，不用完也不会浪费空间。

    • **varchar变长存储相关 **https://liuchenyang0515.blog.csdn.net/article/details/117524328

  • 浮点和日期等数据类型对数据的宽度没有要求一般也不用设定。

开发总结#

  • create_at字段，设置默认值为CURRENT_TIMESTAMP

  • update_at字段设置为DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP，当条目被修改时会自动更新

  • is_del字段，用来表示是否被删除- -

  • id 一般都用自增id作为主键

  • 尽量避免关联查询

  • utf8mb4是utf8的超集，理论上由utf8升级到utf8mb4字符编码没有任何兼容问题。utf8mb4主要是解决了MySQL数据库存储emoji表情的问题。
